# Atlas Change Log

## Purpose
Track meaningful structural changes in the workspace so system evolution stays auditable.

## What to log
Log only high-signal structural actions:
- created notes with clear function
- major rewrites
- merges
- archives
- deletions
- taxonomy moves
- governance/policy changes

## Entry format
- Date: YYYY-MM-DD
- Type: created | modified | moved | merged | archived | deleted | policy
- Path(s): full relevant path(s)
- Why: concise rationale
- Impact: expected effect on clarity/coherence/retrievability
