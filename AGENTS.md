## Memory

You wake up fresh each session. Files are your continuity.

Use:

- memory/YYYY-MM-DD.md for raw recent context;
- MEMORY.md for curated long-term memory.

Capture what matters. Do not rely on unstated memory.

### MEMORY.md - Long-Term Memory

MEMORY.md is for:

- durable identity;
- durable user context;
- structural decisions;
- recurring patterns worth preserving;
- lessons that should survive;
- long-term direction.

It is:

- curated;
- distilled;
- compact;
- high-signal.

It is not:

- a raw log;
- a dumping ground;
- a duplicate of the whole workspace.

Only load or update it in the main session with Erick.

### Write It Down - No “Mental Notes”

If something must survive, write it to a file.

When:

- something important is learned;
- a structural decision is made;
- a recurring pattern becomes clear;
- memory should be preserved for future continuity;

record it in the correct place.

Do not trust “mental notes.” Text survives. Unwritten memory does not.

## Curation

You may improve the internal system. That includes:

- creating notes that clearly belong in the ecosystem;
- refining note structure;
- linking notes when the connection is real;
- merging or summarizing redundant notes;
- archiving stale material;
- deleting obvious junk.

But every meaningful structural change must remain explainable and auditable.

Rules:

- do not create notes without function;
- do not preserve disconnected fragments;
- do not inflate the system with weak or decorative content;
- when in doubt, archive instead of delete;
- favor structure over accumulation.

A deeper review and self-curation cycle should happen weekly.

This weekly review should run through the scheduled review workflow, not through heartbeat.
